Virtual Machine Development

After some research and development I've found it quite annoying to develop on the Virtual Machine. An alternative to this is to use the VS Code extension Live Server (or Five Server if you need PHP). The issue with this is that the file structure is lightly different between the two. The VM is setup to host a server in the 'html' file while Five Server will host in the 'ese_project_vi' file. This makes linking fail between the two because they are being hosted differently.

I'm just about to start working on a solution to this. My plan is to change the VM to host the server in the '/var/www/ese_project_vi/' not the '/var/www/html/ese_project_vi/' folder. This should allow the linking to be the same between the two and won't require developing in a slow VM environment.

Elevator Communication

The car controller was communicating with the elevator controller directly. This needs to change as we need to send the message through the Supervisory Controller. To do this our floor requests need to change our floor requests from 0x05, 0x06, and 0x07 to 0x01, 0x02, 0x03.

Now we are trying to debug the Supervisory Controller's issue. The STM32 is sending the correct message. The Raspberry Pi is receiving the correct message. The Raspberry Pi is not sending a message to the Elevator Controller.

We've confirmed that the Elevator Controller is not the issue because if we load known-working code on the elevator's Pi then the elevator moves just fine.

The Raspberry Pi is able to send a message to the Elevator Controller and it will move to Floor 1 on start-up. Whenever a floor is requested the Elevator Controller doesn't move the car. This should mean that the Raspberry Pi is causing the issue.

We learned that the Car Controller is communicating with the Elevator Controller. We are going to try re-flashing the Elevator Controller to ensure that we know that code running on the controller. Re-flashing the Elevator Controller had no effect on the system.

It seems like it may be an issue with the Supervisory code as all of the messages are being sent but the elevator is still no moving. Every so often the Supervisory controller will receive a request but not do anything with it. It also causes any future requests to be ignored too.

There is an artificial 5 second delay on the Supervisory Controller when a Floor/Car Controller sends a request to allow for time to show that the queue is working with priority.
