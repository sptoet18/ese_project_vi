Besart's Logbook

Week 6

Tasks Completed

CAN Bus Voltage Testing

This week I tested CAN High and CAN Low voltage levels on the elevator system. I used an oscilloscope to capture the CAN waveform and verify that communication activity was present on the bus during elevator operation.

The testing helped confirm that the CAN transceivers were producing measurable differential signals. I also used the oscilloscope captures to better understand how CAN communication appears electrically during system operation.

Elevator Communication Testing

I tested the elevator system while monitoring CAN communication between the controllers. This included observing messages transmitted during elevator calls, floor requests, and system responses.

I also captured UART output from the STM32 microcontrollers. The UART messages made it easier to confirm what commands were being sent and received by the controllers during testing.

Floor Station Wiring

I wired switches and LEDs into the elevator floor stations. The switches were connected to allow user input from the floor stations, while the LEDs were wired to provide visual feedback when buttons were pressed or when the system responded.

After wiring the hardware, I verified that the floor stations were physically connected and ready for further testing with the STM32 controllers.

Request Access Page CSS Development

I worked on the styling for the Request Access page using CSS. This involved improving the layout of the form, styling input fields, formatting fieldsets, and creating a more organized user interface.

I also worked with hover and active states for button styling, which helped me understand how CSS can be used to create interactive visual feedback on a webpage.

Challenges

A challenge this week was capturing reliable CAN signals using the oscilloscope. CAN communication uses differential signaling, so probe placement and grounding were important for getting useful measurements without affecting the circuit.

Another challenge was making sure the hardware wiring, UART output, and CAN communication all matched the expected elevator behavior during testing.

What I Learned

I learned how to use oscilloscope measurements to verify CAN bus activity and how UART output can support debugging by showing controller messages in a readable format. I also learned how CSS can improve a webpage through layout, spacing, input styling, and interactive button effects.

Next Steps

  • Continue validating CAN communication between elevator controllers.
  • Expand testing documentation for CAN and UART verification.
  • Continue improving the Request Access page styling.
  • Perform more integration testing with the wired switches and LEDs.
