Besart's Logbook

Week 10

Tasks Completed

SQL Development Environment Setup

This week I configured the development environment required to work with SQL databases directly through Visual Studio Code on Windows. I installed and configured the necessary VS Code extensions to allow SQL scripts and database queries to be created, viewed, and executed from within the development environment.

This setup will support the database portion of the Raspberry Pi supervisory controller. The database will eventually be used to store elevator events, operating states, commands, and other information received from the elevator control network.

Elevator Code Testing and Troubleshooting

I assisted my project group with testing and troubleshooting the Arduino code used to control the elevator system. This involved observing elevator behaviour during operation and helping determine whether issues were caused by control logic, button inputs, or communication between the controllers.

Repeated testing was performed after software changes were made to confirm that the elevator responded correctly and that the changes did not introduce additional problems. This process helped improve the reliability of the elevator control software.

Open and Close Button Switch Implementation

I helped implement the open and close switch functionality for the elevator buttons. This required assisting with the integration of the physical button inputs into the elevator control logic so that a user could request the elevator doors to open or close.

The button operation was tested to verify that each input was detected correctly and produced the expected response from the elevator controller. This work also required ensuring that the switches operated consistently with the existing elevator control sequence.

Raspberry Pi Website Hosting

I verified that the Raspberry Pi was capable of hosting the project website. The website was deployed to the Raspberry Pi and accessed through the local network using the Raspberry Pi's IP address.

Successfully hosting the website on the Raspberry Pi established the foundation for the supervisory control interface. This interface will eventually allow users to monitor elevator status information and send commands to the elevator system through a web browser.

Challenges

One of the primary challenges this week was troubleshooting the elevator code while multiple hardware and software components were operating together. A problem observed at the elevator could have originated from the original code written for elevator control.

What I Learned

This week I gained additional experience configuring database development tools within Visual Studio Code and preparing SQL functionality for future project integration. I also improved my understanding of how physical button inputs are incorporated into embedded elevator control software.

I learned how a Raspberry Pi can host a website that is accessible over a local network and how this website can serve as the user interface for a larger supervisory control system.

Next Steps

  • Develop code that allows commands to be sent from the hosted website to the Raspberry Pi supervisory controller.
  • Connect website button inputs to the supervisory controller software.
  • Transfer supervisory commands from the Raspberry Pi to the elevator controllers.
  • Configure the SQL database to store elevator commands, status updates, and system events.
  • Test communication between the website, database, Raspberry Pi, and elevator control network.
