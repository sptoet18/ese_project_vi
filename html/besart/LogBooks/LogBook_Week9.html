Besart's Logbook

Week 9

Tasks Completed

Elevator Troubleshooting and Root Cause Analysis

This week I worked closely with my project group members to troubleshoot several issues affecting the elevator's operation. We performed testing of the software to identify the root causes of bugs impacting elevator motion. This involved observing system behaviour, verifying controller operation, and analyzing communication between the microcontrollers during different operating conditions.

Through team debugging sessions, we isolated several software and communication-related issues that contributed to unexpected elevator behaviour. Working through these problems as a team improved both the reliability of the system and our understanding of how each subsystem interacted during operation.

CAN/LAN Communication Preparation

I began preparing the Raspberry Pi supervisory controller for the Phase II implementation of the project. This included configuring the software environment required to host the project website directly from the Raspberry Pi while preparing it to communicate with devices over the Local Area Network (LAN).

I also researched database integration and web server configuration to establish the foundation for the CAN/LAN communication system. This work will allow elevator status information received from the CAN network to be monitored remotely through a web interface hosted on the Raspberry Pi.

Project Integration Support

Throughout the week I continued supporting overall project integration by assisting with testing after software modifications were made. This helped verify that implemented fixes resolved existing issues without introducing new problems and ensured the elevator continued operating reliably during system validation.

I also collaborated with teammates to prepare the project for future Phase II development by ensuring the hardware and software components remained compatible with the upcoming supervisory controller implementation.

Challenges

The primary challenge this week was identifying the source of bugs affecting elevator motion. Since multiple STM32 controllers communicate over the CAN network, determining whether problems originated from software logic, communication, or hardware required careful troubleshooting and repeated testing. Working through these issues highlighted the importance of a structured debugging process when developing distributed embedded systems.

What I Learned

This week I gained additional experience troubleshooting control systems by approaching problems methodically and verifying each subfunction independently before testing the complete system. I also learned more about preparing a Raspberry Pi to host web services and the networking concepts required to support future CAN-to-LAN communication between the elevator controllers and the supervisory interface.

Next Steps

  • Implement CAN/LAN communication using the Raspberry Pi supervisory controller.
  • Configure the Raspberry Pi web server and database for elevator status monitoring.
  • Integrate the supervisory website hosting with the CAN communication network.
